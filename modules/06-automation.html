Module 06

Automation & Toil Reduction

Eliminate repetitive operational work so engineers can focus on improving reliability.

What is Toil?

Google's SRE book defines toil as work that is:

  • Manual — A human must do it
  • Repetitive — Done over and over
  • Automatable — Could be done by a machine
  • Tactical — Reactive, not strategic
  • No enduring value — Doesn't permanently improve the system
  • Scales linearly — Grows with service size
The 50% rule SRE teams should spend no more than 50% of their time on toil. The rest goes to engineering projects that reduce future toil.

Common Sources of Toil

  • Manual deployments and rollbacks
  • Provisioning servers by hand
  • Restarting failed services without auto-healing
  • Responding to the same alert with the same fix
  • Generating reports that could be automated
  • Onboarding new services without templates

Automation Priorities

  1. Auto-remediation — Self-healing for known failure modes (restart, failover, scale).
  2. CI/CD pipelines — Push-button deploys with automated tests and rollbacks.
  3. Infrastructure as Code — Terraform, Pulumi, or CloudFormation for reproducible environments.
  4. Runbooks → Scripts — Every runbook step that repeats should become code.
  5. ChatOps — Trigger common ops tasks from Slack/Teams with guardrails.

Automating Safely

  • Idempotent operations — Running twice produces the same result.
  • Dry-run mode — Preview changes before applying.
  • Audit logs — Record who triggered what and when.
  • Gradual rollout — Canary automation before full deployment.
  • Human-in-the-loop — Require approval for destructive actions.
# Example: auto-restart unhealthy containers
# Kubernetes liveness probe
livenessProbe:
  httpGet:
    path: /health
    port: 8080
  initialDelaySeconds: 15
  periodSeconds: 10
  failureThreshold: 3

Measuring Toil Reduction

Track progress with concrete metrics:

  • Hours/week spent on manual ops tasks (trending down)
  • Number of automated runbook steps vs. manual
  • Mean time to deploy (trending down)
  • Alerts that auto-resolve without human intervention
